# PanelLab 功能构想与路线图

在考虑**用户体验**和**功能完整度**的前提下，将产品构想整理为可落地的功能模块与优先级，与现有 [requirements.md](requirements.md) 及各 [design/*.md](../design/) 衔接。

---

## 1. 构想总览

| 序号 | 模块 | 简述 | 与现有规划关系 |
|------|------|------|----------------|
| 1 | 登录端增强 | 微信/QQ 扫码、手机号验证码登录 | 与 design-login 进阶规划一致，可单独 Plan |
| 2 | 终端增强 | 常用指令库 + 简易 AI 对话接口 | 在现有 design-terminal 基础上扩展 |
| 3 | 用户体验配套 | 服务指南、厂商导航、反馈信箱、账号密码库 | 新增「用户向」功能，可合并为一个 Plan |
| 4 | 多服务器主页 | 地图展示多机分布与状态 | 多机扩展的展示层，依赖多机架构 |
| 5 | 网站与域名管理 | 站点 + 域名绑定、解析、证书等 | 与 design-sites-proxy 衔接并扩展 |
| 6 | （预留） | — | — |
| 7 | 服务器文件管理 | 远程文件浏览、上传下载、编辑 | 新模块，需独立 Plan |

## 2. 分模块说明与实现要点

### 2.1 登录端：微信/QQ 扫码、手机号验证码

| 项目 | 说明 |
|------|------|
| **目标** | 在现有账号密码登录基础上，支持扫码与验证码登录，提升便捷性与安全性。 |
| **必须** | 手机号 + 短信验证码登录（需对接阿里云/腾讯云等短信 API）；微信/QQ 扫码登录（OAuth2 或开放平台扫码）。 |
| **体验** | 登录页提供「密码 / 验证码 / 扫码」Tab 或入口；扫码展示二维码与轮询状态；验证码倒计时、防刷与错误提示。 |
| **依赖** | 短信服务商配置、微信/QQ 应用申请与回调域名；JWT/用户表需支持绑定手机号、openid/unionid。 |
| **文档** | 建议新增 `design-login-oauth-sms.md`，与 [design-login.md](../design/design-login.md) 并列。 |

---

### 2.2 终端：常用指令库 + 简易 AI 对话

| 项目 | 说明 |
|------|------|
| **目标** | 在现有 [网页端 SSH 终端](../design/design-terminal.md) 上增加「常用命令」与「AI 助手」，降低记忆成本、提高操作效率。 |
| **必须** | **常用指令库**：预置/用户可维护的指令列表（如 `df -h`、`systemctl status`、`docker ps`），一键填入终端或执行。可分类：系统、网络、Docker、日志等。 |
| **可选** | **简易 AI 对话**：输入自然语言或简短问题，后端调用大模型 API（如 OpenAI 兼容接口/国产大模型），返回建议命令或简短说明；用户可选择「填入终端」或仅查看。注意：需配置 API Key、考虑流式输出与鉴权。 |
| **体验** | 终端页侧边栏或弹窗展示「常用命令」；AI 区域与终端分离，避免打断当前会话；敏感操作需二次确认。 |
| **依赖** | 现有 WebSocket 终端；可选：大模型 API 配置、流式响应。 |
| **文档** | 建议在 `design-terminal.md` 中增加「常用指令库与 AI 助手」章节，或新建 `design-terminal-commands-ai.md`。 |

---

### 2.3 用户体验：服务指南、厂商导航、反馈信箱、账号密码库

| 项目 | 说明 |
|------|------|
| **目标** | 提升「可发现性」与「可维护性」：新用户能快速上手，运维信息集中管理，问题可反馈。 |
| **必须** | **服务指南**：站内帮助/文档入口（可复用或链接 [user-manual.md](../user/user-manual.md)），结构清晰、支持搜索更佳。**服务器账号密码库**：加密存储各服务器/服务的账号密码（与现有 `monitor_remote_config` 可打通或独立表），列表展示、按服务器/用途分类，支持增删改与「复制密码」；主密钥或解密方式需与当前用户绑定。 |
| **可选** | **服务器提供商网站导航**：常用厂商控制台链接（阿里云、腾讯云、AWS、Vultr 等）集中展示，可配置自定义链接。**反馈信箱**：简单表单（内容 + 联系方式）提交到后端，落库或发邮件；或仅提供邮箱/链接。 |
| **体验** | 导航栏或仪表盘提供「帮助」「导航」「反馈」入口；密码库需鉴权、脱敏展示、操作日志可选。 |
| **依赖** | 用户体系、可选邮件服务；密码库需加密方案（如与现有 utils.cipher 统一）。 |
| **文档** | 建议新建 `design-ux-guides-vault.md`（服务指南 + 厂商导航 + 反馈 + 密码库）。 |

---

### 2.4 多服务器管理主页：地图显示

| 项目 | 说明 |
|------|------|
| **目标** | 多机场景下，在主页用地图展示各服务器地理位置与状态，一目了然。 |
| **必须** | 地图组件（如 Leaflet/高德/百度地图）；每个服务器可配置经纬度或通过 IP 解析大致位置；展示在线/离线、告警等状态；点击可进入该机详情或监控。 |
| **可选** | 地域/机房分组、链路示意、聚合展示。 |
| **依赖** | 先有多服务器架构（面板中台 + 被控机 Agent 或 SSH 配置列表）；服务器表需有地理位置或 IP。 |
| **文档** | 建议在多服务器架构设计完成后，新建 `design-multiserver-dashboard.md`，其中一节为「地图展示」。 |

---

### 2.5 网站与域名管理

| 项目 | 说明 |
|------|------|
| **目标** | 在 [站点与反向代理](../design/design-sites-proxy.md) 基础上，增加「域名」维度：绑定、解析提示、证书申请/续期。 |
| **必须** | 站点与域名绑定（一对多）；域名列表与站点关联展示；证书管理（如 Let's Encrypt 申请与自动续期）。 |
| **可选** | 解析记录说明/跳转厂商 DNS 控制台；多服务器下按机器维度管理站点与域名。 |
| **依赖** | design-sites-proxy 已实现站点与 Nginx；需 ACME 客户端或调用 certbot 等。 |
| **文档** | 可在 `design-sites-proxy.md` 中扩展「域名与证书」章节，或新建 `design-sites-domain-ssl.md`。 |

### 2.7 服务器文件管理

| 项目 | 说明 |
|------|------|
| **目标** | 在浏览器中浏览远程服务器目录、上传/下载文件、简单文本编辑，减少对 SFTP 客户端的依赖。 |
| **必须** | 目录树与文件列表；上传、下载；文本文件在线编辑与保存；路径与权限校验。 |
| **可选** | 重命名、删除、新建目录；压缩/解压；多选操作；大文件分片上传。 |
| **依赖** | 现有 SSH/paramiko 或独立 SFTP 通道；多服务器时按机器切换。 |
| **安全** | 限制可访问路径（如白名单）、禁止执行；敏感文件脱敏或拦截。 |
| **文档** | 建议新建 `design-filemanager.md`。 |

### 2.8 服务器面板：流量监控与访问记录统计

| 项目 | 说明 |
|------|------|
| **目标** | 在 [系统监控](../design/design-monitor.md) 基础上，增加「流量」与「访问记录」维度，便于分析带宽与访问来源。 |
| **必须** | **流量监控**：按网卡/时间粒度的入站/出站流量统计与图表（可与现有监控合并展示）。**访问记录统计**：解析 Nginx/Apache 访问日志，统计 UV/PV、状态码、路径、IP/地域等，简单图表与列表。 |
| **可选** | 多机汇总、告警阈值、导出报表。 |
| **依赖** | 本机/远程监控数据；日志路径配置与解析（如正则或 goaccess 等）。 |
| **文档** | 可在 `design-monitor.md` 或 `design-monitor-remote.md` 中扩展「流量与访问统计」，或新建 `design-monitor-traffic-access.md`。 |

## 3. 建议实现顺序（兼顾体验与完整度）

在现有阶段 0～5 的基础上，建议扩展顺序如下（可与课程/时间安排灵活调整）：

| 阶段 | 内容 | 说明 |
|------|------|------|
| 现有 1～4 | 登录、仪表盘、监控、站点与反向代理、数据库 | 保持为主干，优先完成 MVP。 |
| **A** | 终端增强：常用指令库 | 与现有终端结合紧，收益高，先做指令库再考虑 AI。 |
| **B** | 用户体验：服务指南 + 密码库 | 提升日常可用性；厂商导航与反馈可一并做小版本。 |
| **C** | 网站与域名管理（域名 + 证书） | 与阶段 3 站点功能衔接自然。 |
| **D** | 服务器文件管理 | 独立模块，依赖 SSH/SFTP，多机稳定后再做更顺。 |
| **E** | 流量监控 + 访问记录统计 | 监控的延伸，数据来源明确。 |
| **F** | 多服务器架构 + 地图主页 | 依赖多机中台与 Agent，放在后端架构扩展之后。 |
| **G** | 登录端：验证码 + 微信/QQ | 依赖第三方配置，可在用户体系稳定后做。 |
| **H** | 终端：简易 AI 对话 | 在指令库与 API 配置就绪后作为增强。 |
