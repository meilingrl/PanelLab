# PanelLab 用户使用说明书

本文档面向使用 PanelLab 面板的最终用户，说明如何登录、使用各功能模块及常见问题处理。

---

## 1. 产品简介

**PanelLab** 是一款轻量级服务器运维管理面板，可用于：

- 查看本机或远程 Linux 服务器的 CPU、内存、磁盘、网络等系统状态
- 在网页中通过 SSH 连接远程服务器终端，执行命令（无需本地 PowerShell 等）
- 管理静态站点与 Nginx 反向代理（创建站点、编辑、生效、删除）
- 后续将支持数据库（MySQL）管理、计划任务等

**适用对象**：服务器管理员、运维人员；支持在浏览器中完成常用运维操作，无需频繁登录 SSH。

**访问方式**：部署完成后，在浏览器中打开面板地址（如 `http://你的服务器:8000` 或 `http://localhost:5173`，具体以部署说明为准）。

## 2. 登录与账号

### 2.1 登录

1. 打开面板登录页，输入**用户名**和**密码**。
2. 点击「登录」。
3. 登录成功后会自动进入**用户中心**页。

若用户名或密码错误，页面会提示「用户名或密码错误」或接口返回的具体信息。

**首次使用**：若管理员已执行过数据库初始化（`python -m init_db`），默认管理员账号为：

- 用户名：`admin`
- 密码：由部署时的 `INIT_ADMIN_PASSWORD` 环境变量决定（默认多为 `admin`）

建议首次登录后立即**修改密码**（见下文）。

### 2.2 注册新账号

1. 在登录页点击「没有账号？注册」。
2. 填写用户名（至少 2 个字符）、密码（至少 6 位）、确认密码。
3. 点击「注册」。
4. 注册成功后可使用新账号登录。

### 2.3 修改密码

1. 登录后，在左侧边栏底部点击「**修改密码**」。
2. 在弹窗中填写：**原密码**、**新密码**、**确认新密码**。
3. 新密码至少 6 位，且不能与原密码相同。
4. 点击「确定」完成修改，弹窗关闭。

### 2.4 退出登录

在左侧边栏底部点击「**退出**」，将清除登录状态并跳转到登录页。

## 3. 界面与导航

### 3.1 整体布局

- **左侧边栏**：PanelLab  logo、功能导航、当前用户名、主题切换、修改密码、退出。
- **主内容区**：根据左侧所选菜单显示对应页面。

### 3.2 功能菜单

| 菜单       | 说明 |
|------------|------|
| 用户中心   | 服务器数据中心（多台服务器 SSH 信息）、账号与安全（修改密码）、快捷帮助。 |
| 系统监控   | 本机/远程监控、CPU/内存/磁盘圆环、网络流量图、进程列表。 |

### 3.3 主题切换

在登录页或仪表盘左侧边栏点击 **🌙 / ☀️** 按钮，可切换深色/浅色主题，偏好会保存在浏览器本地。

## 4. 用户中心

用户中心集中管理账号与服务器信息，左侧菜单置顶便于访问。

### 4.1 服务器数据中心

- **作用**：维护多台服务器的 SSH 连接信息（名称、IP/主机名、端口、用户名、密码），供「终端控制」页新建终端时选择。密码加密存储，仅当前用户可见。
- **操作**：
  - **新建服务器**：点击「新建服务器」，填写名称、IP/主机名（如 `192.168.0.10`）、端口（默认 22）、用户名、密码，保存。
  - **编辑**：在列表中点「编辑」可修改该服务器信息；修改密码时填写新密码，不修改则留空。
  - **删除**：点「删除」并确认后，该服务器从列表中移除；若终端中有该服务器的会话，关闭对应标签即可。
- 至少添加一台服务器后，才能在「终端控制」中新建终端连接。

### 4.2 账号与安全

- 修改密码：与左侧边栏底部「修改密码」一致，见 [2.3 修改密码](#23-修改密码)。

### 4.3 快捷帮助

- 可折叠区块，提供常用说明或链接。

## 5. 系统监控

系统监控页用于查看本机或已配置的远程 Linux 主机的资源使用情况。

### 5.1 选择监控目标

- 页面上方有「本机」或「远程」等目标选项，选择后下方数据会切换为对应主机的指标。
- 若配置了远程主机，选择远程目标后可查看该主机的 CPU、内存、磁盘、网络等。

### 5.2 本机监控内容

- **圆环图**：CPU 使用率、内存使用率、磁盘使用率，以百分比显示。
- **网络流量图**：入站/出站流量随时间变化的曲线（若接口提供历史数据）。
- **进程列表**：当前主机上的进程列表，支持按 CPU、内存等排序，可设置显示条数、按名称过滤。
- 数据约每 5 秒自动刷新一次。

### 5.3 远程监控配置

若要监控另一台 Linux 服务器：

1. 在监控页找到「远程配置」或「配置远程主机」入口。
2. 填写远程主机的 **IP/主机名**、**SSH 端口**（默认 22）、**登录用户名**、**密码**（或密钥路径，视界面而定）。
3. 保存后，在监控目标中选择该远程主机即可查看其指标。
4. 若远程机未安装 Python 或 `psutil`，面板可能提供「安装 psutil」等按钮，按提示在远程机上完成依赖安装后再查看。

**注意**：远程功能依赖后端 SSH 配置（如 `REMOTE_SSH_*` 环境变量），若未配置或连接失败，页面会给出相应提示。

## 6. 远程终端（终端控制）

远程终端可在浏览器中直接打开远程服务器的 Shell，支持多终端标签、持久连接与常用指令，无需在本地使用 PowerShell 或其它终端工具。

### 6.1 使用前准备

- 须先在 **用户中心 → 服务器数据中心** 中至少添加一台服务器（IP/主机名、端口、用户名、密码）；终端从该列表中选择要连接的服务器。

### 6.2 连接与使用

1. 登录后，在左侧菜单点击「**终端控制**」进入终端页。
2. 若尚未有终端会话且已添加过服务器，可能会自动打开第一个服务器的终端；也可点击「**+ 新建终端**」，在弹窗中从服务器库选择一台服务器，确认后建立新会话。
3. 每个会话以**标签**形式显示在顶部，标签名为该服务器的 IP（或主机名）；**绿点**表示已连接，黄点表示连接中，红点表示断开或错误。
4. 在终端区域输入命令即可在远程服务器上执行；切换其他菜单再返回终端页，连接会保持，无需重连。
5. 关闭某个标签（标签上的 **×**，悬停可见）即断开该终端会话；关闭所有标签后，可随时再点「+ 新建终端」新建会话。

### 6.3 常用指令与历史

- 终端页**右侧栏**提供「常用指令」（系统、网络、Docker、日志等），点击即可向**当前选中的终端**发送对应命令。
- 「历史指令」自动记录本机曾发送过的命令，点击可再次发送；支持「清空」历史。

### 6.4 界面说明

- 终端区域为浅色面板风格，与系统监控等页面一致；窗口大小变化时终端行列自动适配。
- 若未添加服务器，页面会提示「请先在用户中心 → 服务器数据中心添加服务器」。

### 6.5 常见问题

- **提示需在用户中心添加服务器**：请先在用户中心 → 服务器数据中心中新建至少一台服务器。
- **SSH 连接失败**：请检查网络是否可达、远程是否开启 SSH、用户名与密码是否正确；可在用户中心编辑该服务器信息或本地用 `ssh 用户@主机` 测试。
- **连接后马上断开**：可能是登录态过期，请重新登录后再试；生产环境需确认 Nginx 等已正确代理 WebSocket（`/api`）。

## 7. 网站与反向代理

本模块用于管理 Nginx 的站点配置：添加静态站点或反向代理站点，并尝试将配置写入 Nginx、校验并重载。

### 7.1 站点列表

- 列表展示所有已添加的站点，包括：**名称**、**域名**、**类型**（静态站点 / 反向代理）、**根目录或代理目标**、**端口**、**状态**、**是否启用**、**更新时间**、**操作**。
- **状态**说明：
  - **已生效**：配置已成功写入并完成 Nginx 校验与重载。
  - **未生效**：仅保存记录，尚未成功生效或未点击「生效」。
  - **错误**：上次生效失败（如 Nginx 未安装、配置校验失败、重载失败），页面或接口会返回具体错误信息。

### 7.2 新增站点

1. 点击右上角「**新增站点**」。
2. 在下方表单中填写：
   - **站点名称**：面板内标识，建议唯一。
   - **域名**：对外访问的域名（如 `blog.example.com`）。
   - **类型**：选择「静态站点」或「反向代理」。
   - **监听端口**：默认 80，可按需修改（1–65535）。
   - **启用站点**：勾选表示启用，不勾选则不会生成启用状态的 Nginx 配置。
   - **根目录**（仅静态站点）：静态文件所在路径，如 `/var/www/blog`。
   - **代理目标地址**（仅反向代理）：后端服务地址，如 `http://127.0.0.1:3000`，须为 `http://` 或 `https://` 开头。
3. 点击「**保存并生效**」。
4. 若保存成功，会尝试生成 Nginx 配置、执行 `nginx -t` 校验并重载 Nginx；结果通过成功/错误提示反馈。若环境未安装 Nginx 或未配置写权限，可能提示「未找到命令 nginx」等，此时仅保存了站点记录，状态可能为「错误」。

### 7.3 编辑站点

1. 在列表中找到对应站点，点击「**编辑**」。
2. 修改站点名称、域名、类型、端口、根目录/代理目标、启用状态等。
3. 点击「**保存并生效**」，逻辑同新增：先更新记录，再尝试 Nginx 生效。

### 7.4 手动生效

若某站点状态为「未生效」或「错误」，可在修改环境（如安装好 Nginx、修正配置目录权限）后，对该站点点击「**生效**」，会重新生成配置并执行校验与重载，无需再编辑保存。

### 7.5 删除站点

1. 在列表中点击该站点的「**删除**」。
2. 在确认框中确认删除（会提示将影响系统 Nginx 配置）。
3. 确认后，面板会删除该站点记录，并尝试从 Nginx 配置目录中移除对应配置文件、执行校验与重载。

### 7.6 常见错误提示

| 提示或现象 | 可能原因 | 建议 |
|------------|----------|------|
| Nginx 配置校验失败：未找到命令 `nginx` | 系统未安装 Nginx 或未加入 PATH | 在目标 Linux 上安装 Nginx，或将面板设为跳过 Nginx 生效（如开发环境设置 `NGINX_SKIP_APPLY=true`） |
| 配置目录不可写、重载失败等 | 权限或路径配置问题 | 检查后端配置的 Nginx 配置目录及进程权限 |
| 域名已存在 / 站点名称已存在 | 与已有站点冲突 | 更换域名或站点名称 |
| 静态站点必须填写根目录 / 反向代理必须填写代理目标 | 表单校验 | 按类型填写必填项，代理目标须为合法 http(s) URL |

## 9. 常见问题

### 9.1 登录后刷新页面又要重新登录

部分部署方式下，前端与后端分离（如前端 `npm run dev` 单独端口），若未正确携带 Cookie 或 Token，可能被判定未登录。请确认：

- 登录后请求是否携带 `Authorization: Bearer <token>`（Token 通常保存在浏览器本地）。
- 若使用 Nginx 反向代理，是否将 `/api` 正确转发到后端，且未丢失 Header。

### 9.2 监控数据不更新或一直「—」

- 本机监控：确认后端有权限读取系统指标（如 psutil），且接口 `/api/monitor/stats?target=local` 返回正常。
- 远程监控：确认远程 SSH 配置正确、网络可达，且远程机已安装所需依赖（如 psutil）。

### 9.3 站点保存成功但状态为「错误」

多数为 Nginx 未安装、未在 PATH 中、或配置目录无写权限。请查看页面/接口返回的 `detail` 错误信息，按提示在服务器上安装 Nginx、配置目录权限，或使用「生效」按钮重新尝试。

### 9.4 修改密码后无法登录

请使用**新密码**登录。若忘记密码，需由管理员在服务器上通过后端脚本重置（如项目提供的 `change_password.py` 等，具体见项目文档）。

### 9.5 终端连接失败或立即断开

- 若提示需在用户中心添加服务器：请先在 **用户中心 → 服务器数据中心** 添加至少一台服务器。
- 若提示「SSH Connection failed」或「Server not found」：检查该服务器 IP、端口、用户名与密码是否正确；可先在本地终端用 `ssh 用户@主机` 测试。
- 若 WebSocket 连接失败或连接后立即断开：请确认已登录且未过期；生产环境需确保 Nginx 等已正确代理 `/api` 的 WebSocket。
