# 仪表盘：需求与设计（阶段 1）

本文档明确仪表盘的范围、布局与多界面骨架，为后续系统监控、网站与反向代理、数据库管理等留好空间。

---

## 1. 需求范围（本阶段）

| 项目 | 说明 |
|------|------|
| 仪表盘定位 | 登录后的主工作区：系统概览、快捷入口、基础状态展示（见 requirements 2.2） |
| 本阶段目标 | 完成**骨架**：统一布局、侧边/顶导航、多子页面占位，内容后续阶段填充 |
| 多界面 | 为「系统监控」「网站与反向代理」「数据库管理」等预留路由与空白页，避免后续大改 |

---

## 2. 布局设计

- **整体**：左侧固定侧边栏（导航 + 用户区）+ 右侧主内容区（`<router-view>`）。
- **侧边栏**：
  - 顶部：Logo / 产品名（可点击回概览）。
  - 导航项：概览、系统监控、网站与反向代理、数据库管理（后续可加：计划任务、设置等）。
  - 底部：当前用户、主题切换、修改密码、退出。
- **主内容区**：按路由渲染子页面，默认为「概览」页。

---

## 3. 路由与页面骨架

| 路径 | 名称 | 说明 |
|------|------|------|
| `/` 或 `/dashboard` | 概览 | 仪表盘首页，后续：系统概览、快捷入口、基础状态 |
| `/monitor` | 系统监控 | 占位，阶段 2 接入 CPU/内存/磁盘/网络等 |
| `/sites` | 网站与反向代理 | 占位，阶段 3 接入站点与 Nginx |
| `/databases` | 数据库管理 | 占位，阶段 4 接入 MySQL 等 |

所有上述路由均需登录（`requiresAuth: true`）。布局组件（DashboardLayout）作为父级，子路由在其内渲染。

---

## 4. 前端实现要点

- **布局组件**：`DashboardLayout.vue` — 侧边栏 + header（可选）+ `<router-view>`。
- **子页面**：`views/dashboard/Overview.vue`、`Monitor.vue`、`Sites.vue`、`Databases.vue`，本阶段仅标题 + 简短占位文案。
- **路由**：父路由 `/` 使用 `component: DashboardLayout`，`children` 为上述子路由；默认重定向到 `/dashboard` 或子路径「概览」。
- **样式**：与现有主题变量一致，侧边栏可折叠（可选，本阶段可先固定宽度）。

---

## 5. 与后续阶段的衔接

- 阶段 2：在 `/monitor` 页接入系统监控 API 与图表。
- 阶段 3：在 `/sites` 页接入站点列表、添加/编辑、Nginx 配置。
- 阶段 4：在 `/databases` 页接入数据库列表与管理。
- 用户相关：修改密码、注册入口放在布局内（用户菜单或独立设置页），与 `design-login.md` 及用户体系扩展一致。
