# 系统监控页 UI 与能力增强：需求与设计（Plan）

本文档为系统监控页的三项增强：圆环百分比展示、流量曲线图、远程一键安装 psutil。格式与 [docs/README.md](../README.md) 中「Plan 格式说明」一致；依赖 [design-monitor.md](design-monitor.md)、[design-monitor-remote.md](design-monitor-remote.md)。

---

## 1. 目标与范围

| 项目 | 说明 |
|------|------|
| 目标 | 提升监控页可视化与实用性：百分比用圆环展示更直观；网络增加流量速率与平滑曲线图；远程监控支持一键在目标机安装 psutil。 |
| 本阶段范围 | **圆环**：CPU、内存、各磁盘使用率改为环形仪表盘（保留数值与简要文字）；**流量图**：本机网络收发速率、最近一段时间平滑曲线；**远程安装**：提供「在远程主机安装 psutil」按钮，后端可先占位再实现真实 SSH 执行。 |
| 不包含 | 远程主机的流量曲线（可后续扩展）；进程结束等写操作。 |
| 依赖/前置 | 现有 `/api/monitor/stats`、远程配置与 SSH 采集链路；前端 Monitor.vue 与主题变量。 |

## 2. 功能需求

### 2.1 圆环百分比

| 项目 | 说明 |
|------|------|
| 展示对象 | CPU 使用率、内存使用率、各磁盘使用率。 |
| 形式 | 环形（圆环）仪表盘，圆心或环内显示百分比数字；保留「已用/总量」等简要文字说明。 |
| 交互 | 仅展示，无点击交互；与现有 5 秒轮询数据一致。 |
| 风格 | 使用现有 CSS 变量（如 `--accent`、`--border`），明暗主题一致。 |

### 2.2 流量监控与平滑曲线图

| 项目 | 说明 |
|------|------|
| 数据 | 本机网络收发**速率**（如 KB/s 或 MB/s），以及最近一段时间（如 2–5 分钟）的**历史序列**，用于绘制曲线。 |
| 展示 | 平滑曲线图：横轴时间，纵轴速率；可两条线（接收/发送）或双 Y 轴；曲线平滑（如 tension/bezier）。 |
| 范围 | 先做**本机**；远程目标机暂无历史与曲线，可仅显示当前速率或累计量。 |
| 刷新 | 与现有 stats 轮询一致（如 5 秒）；曲线随新数据点追加，保留固定长度（如 60 点）。 |

### 2.3 远程一键安装 psutil

| 项目 | 说明 |
|------|------|
| 入口 | 在「远程服务器连接」区块或监控目标为「远程」时，提供按钮「在远程主机安装 psutil」。 |
| 行为 | 点击后调用后端接口；若后端已实现则通过 SSH 在目标机执行安装命令（如 `pip3 install --user psutil`），返回成功或失败原因；若后端未实现则返回 501/占位文案，前端提示「功能开发中，请手动在远程执行: pip3 install --user psutil」。 |
| 安全 | 仅对已保存的远程配置使用；不暴露密码；安装命令只读/安装包，不执行任意脚本。 |

## 3. 接口约定

### 3.1 系统快照扩展（GET /api/monitor/stats）

在现有响应基础上，**本机**（target=local）时可增加以下字段（与现有字段兼容，不破坏旧前端）：

- **network** 扩展（可选）：
  - `bytes_sent`、`bytes_recv`：保持为累计值。
  - `rate_sent_kbps`、`rate_recv_kbps`：当前瞬时速率（千字节/秒），由后端两次采样间隔计算或内存中最近两次累计值差分得到。
- **network_history**（可选）：最近 N 个采样点的历史，用于前端画曲线。每项格式如：`{ "ts":  Unix 秒, "rate_sent_kbps": 12.5, "rate_recv_kbps": 88.2 }`，数量由后端决定（如 60 点，每 5 秒一点约 5 分钟）。

若实现简单，可仅返回 `rate_sent_kbps` / `rate_recv_kbps`，由**前端**在轮询时自行维护最近若干次采样并计算速率或直接存点画图（避免后端状态）。

### 3.2 远程安装 psutil（POST /api/monitor/remote-install-psutil）

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | /api/monitor/remote-install-psutil | 需登录；使用当前保存的远程配置，通过 SSH 在目标机执行安装 psutil 的命令；若未配置远程或未实现则返回 503/501。 |

**响应示例（成功）：**

```json
{ "ok": true, "message": "已在远程主机成功安装 psutil" }
```

**响应示例（失败）：**

```json
{ "ok": false, "detail": "远程执行失败：pip 未找到" }
```

**占位实现（未开发时）：** 返回 HTTP 501，body：`{ "detail": "功能开发中，请在远程主机手动执行: pip3 install --user psutil" }`。

## 4. 数据与存储

- **本机网络历史**：仅用于曲线图，可放在**后端内存**中（如按 target=local 键的 deque，最多 60 条），不落库；进程重启后清空。
- **远程配置**：沿用现有 `monitor_remote_config` 表，无新增存储。
- 无新增数据库表。

---

## 5. 后端实现要点

- **速率/历史**：本机在 `_get_network()` 或 stats 处理中，保存上一时刻的 `bytes_sent`/`bytes_recv` 与时间戳，本次减上次除以间隔得到 rate；或将每次 stats 的 network 与 ts 写入内存 deque，返回最近 60 条给前端。注意仅 target=local 时计算与返回。
- **remote-install-psutil**：新路由，依赖 `get_current_user`；读取当前远程配置并解密密码，用 paramiko 连接后执行 `python3 -m pip install --user psutil` 或 `pip3 install --user psutil`，捕获 stdout/stderr，解析是否成功；若未配置返回 503，若暂不实现则直接返回 501 与占位文案。
- **错误处理**：SSH 超时、命令不存在、权限不足等返回 4xx/5xx 与可读 detail。

---

## 6. 前端实现要点

- **圆环**：用 SVG 或 CSS 绘制圆环（如 `stroke-dasharray` / `stroke-dashoffset` 表示百分比），中间文字显示百分比；替换现有 CPU、内存、磁盘的条形进度块，保留「总/已用/可用」等文字。
- **流量图**：使用轻量图表库（如 Chart.js）或 SVG path 自绘平滑曲线；数据源为 stats 中 `network_history` 或前端根据多次 `network` 采样自算速率并维护数组；横轴时间，纵轴 KB/s 或 MB/s；仅本机展示曲线，远程可仅显示当前速率或累计。
- **安装 psutil 按钮**：在远程配置区块或监控目标为远程时的显眼位置放置按钮，调用 `POST /api/monitor/remote-install-psutil`，根据 501/503/200 显示不同提示（开发中 / 未配置 / 成功或失败原因）。
- **主题**：圆环与图表颜色使用 CSS 变量，与现有主题一致。

---

## 7. 与后续阶段衔接

- 圆环与流量图仅依赖现有 stats 与可选扩展字段，不影响阶段 3（网站与反向代理）。
- 远程安装 psutil 属于监控能力扩展，不依赖反向代理；若与阶段 3 排期冲突，可长期保留 501 占位。
