# 数据库与计划任务：需求与设计（阶段 4）

本文档为阶段 4「数据库与计划任务」的 Plan，格式与 [docs/README.md](../README.md) 中「Plan 格式说明」一致。

---

## 1. 目标与范围

| 项目 | 说明 |
|------|------|
| 目标 | 在仪表盘「数据库管理」页完成 MySQL 基础管理闭环（库/用户列表、创建、删除），并给出计划任务（cron）的最小可落地方案。**管理对象为用户所选「要管理的服务器」上的 MySQL**，而非面板所在机或单一固定实例。 |
| 本阶段范围 | **包含**：按服务器维度的数据库与用户 CRUD、危险操作确认、执行结果反馈、基础权限与参数校验；**不包含**：SQL 在线编辑器、跨实例复制、备份恢复平台化、复杂任务编排。 |
| 运行环境 | 目标运行环境为 **Linux + MySQL**；Windows 开发阶段可先完成前后端接口与交互，最终必须在 Linux 上验证真实命令执行。数据库管理针对**用户所选远程服务器**上的 MySQL（一户多机场景）。 |
| 依赖/前置 | 依赖阶段 1 登录鉴权与用户数据隔离、阶段 3 页面风格规范与错误反馈模式、**一户多机（Server 模型）**：需有「当前要管理的服务器」概念，以便连接该机上的 MySQL。若需在目标机执行系统命令，需确认运行账号具备最小必要权限。 |

## 2. 功能需求

### 2.1 必须功能（MVP）

| 功能 | 说明 |
|------|------|
| 目标服务器选择 | 用户在前端选择「当前要管理的服务器」；数据库列表、用户列表及后续创建/删除操作均针对**该服务器上的 MySQL 实例**。未选服务器或该服务器未配置 MySQL 时，需明确提示。 |
| 数据库列表 | 返回**当前选中服务器**上可管理数据库列表（可按业务约定仅展示非系统库），含创建时间等基础信息。 |
| 创建数据库 | 在选中服务器的 MySQL 上创建数据库，字符集默认 `utf8mb4`。 |
| 删除数据库 | 在选中服务器上删除指定数据库（默认要求二次确认，提示不可逆）。 |
| 用户列表 | 返回**当前选中服务器**上 MySQL 的数据库用户列表（至少包含用户名、主机、权限摘要）。 |
| 创建用户并授权 | 在选中服务器的 MySQL 上创建用户并授予指定数据库权限（默认最小可用权限策略）。 |
| 删除用户 | 在选中服务器上删除指定数据库用户并提示风险。 |
| 错误反馈 | 连接失败、权限不足、SQL 执行失败、未选服务器等错误必须明确返回给前端。 |

### 2.2 阶段内可选（建议）

- 数据库详情页（库大小、表数量统计）
- 用户权限编辑（变更授权）
- 连接可用性检测按钮（一键校验 MySQL 连接）

### 2.3 计划任务（cron）范围约束

本阶段只做“可选预研 + 结构预留”，不强制进入 MVP：

## 3. 接口约定

所有接口均要求登录（Bearer JWT），统一前缀为 `/api/db`。**数据库与用户管理均需指定目标服务器**，以便连接该服务器上的 MySQL。

### 3.1 数据库管理

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | /api/db/databases | 返回数据库列表。**Query：`server_id`**（必填），指定要管理的服务器（当前用户名下）。 |
| POST | /api/db/databases | 创建数据库。**Body 或 Query：`server_id`**（必填）。 |
| DELETE | /api/db/databases/{name} | 删除数据库。**Query：`server_id`**（必填）。 |

**创建数据库请求示例：**

```json
{
  "name": "panel_app",
  "charset": "utf8mb4",
  "collation": "utf8mb4_unicode_ci"
}
```

### 3.2 用户管理

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | /api/db/users | 返回**当前选中服务器**上 MySQL 的用户列表。**Query：`server_id`**（必填）。 |
| POST | /api/db/users | 在选中服务器的 MySQL 上创建用户并授权。**Body 或 Query：`server_id`**（必填）。 |
| DELETE | /api/db/users/{username} | 在选中服务器上删除用户（可通过 query 指定 host）。**Query：`server_id`**（必填）。 |

**创建用户请求示例：**

```json
{
  "username": "panel_reader",
  "password": "StrongPassword_123",
  "host": "%",
  "database": "panel_app",
  "privileges": ["SELECT", "INSERT", "UPDATE", "DELETE"]
}
```

### 3.3 计划任务（预留接口）

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | /api/db/jobs | 返回计划任务列表（预留，可返回空实现）。 |
| POST | /api/db/jobs | 新建计划任务（预留）。 |
| DELETE | /api/db/jobs/{id} | 删除计划任务（预留）。 |

### 3.4 错误约定

- `400`：参数非法（名称不合法、密码过弱、权限列表为空）
- `401`：未登录或 token 无效
- `403`：当前用户无操作权限
- `404`：目标数据库或用户不存在
- `409`：数据库名或用户名冲突
- `500`：服务端内部错误
- `503`：MySQL 不可用或命令执行失败

## 4. 数据与存储

### 4.1 阶段四数据策略

阶段四以“管理已有 MySQL 实例”为主，业务数据主要来自 MySQL 元数据，不强制新增业务表。

### 4.2 计划任务预留表（可选）

若决定在本阶段先落库，可新增 `scheduled_jobs`：

| 字段 | 说明 |
|------|------|
| id | 主键 |
| name | 任务名称 |
| cron_expr | cron 表达式 |
| command | 执行命令 |
| enabled | 是否启用 |
| last_run_at | 最近执行时间 |
| last_status | 最近执行状态 |
| created_at | 创建时间 |
| updated_at | 更新时间 |

### 4.3 配置来源与目标实例

- **当前实现（过渡）**：可继续使用环境变量 `MYSQL_*` 连接**单一 MySQL 实例**（如面板所在机或本机），用于开发或“仅管理本机 MySQL”场景。
- **目标行为（一户多机）**：数据库管理针对**用户所选服务器**上的 MySQL。连接信息来自：
  - 该服务器与用户关联（`Server` 表，当前用户名下）；
  - 该服务器上的 MySQL 地址与凭证：可为「与 SSH 同机、默认 3306」或在该服务器上单独配置（如扩展 `Server` 表字段：`mysql_host`、`mysql_port`、`mysql_admin_user`、`mysql_admin_password_encrypted`，或约定使用 SSH 隧道/代理访问该机 MySQL）。
- 明确区分“面板自身业务连接库”和“被管理的 MySQL 实例”，避免误删当前业务核心库。

## 5. 后端实现要点

### 5.1 模块划分建议

- `routers/db_admin.py`：数据库与用户接口
- `schemas/db_admin.py`：请求/响应模型
- `services/mysql_admin.py`：MySQL 管理命令与 SQL 执行封装
- （可选）`routers/jobs.py` + `services/cron_manager.py`：计划任务预留

### 5.2 实现策略

- **按 server_id 解析目标 MySQL**：router 校验 `server_id` 属于当前用户后，根据该 Server 解析 MySQL 连接信息（或建立到该机的连接/隧道），再调用 `mysql_admin` 在该连接上执行 list/create/delete。
- 优先采用参数化 SQL，避免字符串拼接导致注入风险。
- 将“SQL 执行/异常映射”集中在 service 层，router 仅做入参校验、server 归属校验与响应转换。
- 统一返回可读错误信息（含“未选择服务器”“该服务器未配置 MySQL”“连接超时”等），便于前端直接提示。

### 5.3 安全注意点

- 严格限制可操作对象：默认禁止删除系统库（如 `mysql`、`information_schema`）
- 用户名/库名采用白名单字符校验（字母、数字、下划线）
- 危险操作必须要求前端显式确认（例如 `confirm_text`）
- 避免日志打印明文密码

## 6. 前端实现要点

### 6.1 页面结构（`/databases`）

至少包含：

- **服务器选择**：下拉或列表选择「当前要管理的服务器」（仅展示当前用户名下的服务器）；未选时列表区提示“请先选择要管理的服务器”，且不请求库/用户列表。
- 数据库列表卡片 + 创建按钮（针对选中服务器）
- 数据库用户列表卡片 + 创建按钮（针对选中服务器）
- 新建弹层（数据库/用户）
- 删除二次确认弹窗
- 统一成功/错误提示

### 6.2 交互规范

- 沿用阶段 3 的交互模式：列表加载态、空态、提交禁用态
- 危险操作按钮使用高风险视觉样式
- 后端返回 `detail` 时优先展示 `detail`

### 6.3 与现有风格一致性

- 保持与 `Sites.vue`、`Monitor.vue` 相同的卡片、按钮、表格体系
- 避免在阶段四引入新的大型 UI 依赖

## 7. 与后续阶段衔接

- **一户多机**：数据库管理按「当前选中的服务器」操作该机 MySQL，与终端、监控的“选服务器”体验一致；依赖 `Server` 模型及可选的 MySQL 连接信息扩展。
- 阶段 5 可在此基础上扩展为“数据库备份与恢复”“慢查询诊断”“安全审计”
- 计划任务若在本阶段仅预留，可在后续直接接续接口与页面骨架；计划任务亦可按服务器维度执行（在选中服务器上管理 cron）。
- 多机扩展时复用“按 server_id 连接目标机”的思路，数据库管理、计划任务均可扩展到用户名下的多台主机。
