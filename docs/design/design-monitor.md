# 系统监控：需求与设计（阶段 2）

本文档为阶段 2 本机系统监控的 Plan，格式与 [docs/README.md](../README.md) 中「Plan 格式说明」一致。

---

## 1. 目标与范围

| 项目 | 说明 |
|------|------|
| 目标 | 在仪表盘「系统监控」页展示当前主机的 CPU、内存、磁盘、网络等基础指标，便于日常运维查看。 |
| 本阶段范围 | **监控与展示**：实时/准实时数据、卡片或列表展示、可选简单图表；**进程列表**（只读）已在本阶段扩展实现；结束进程等写操作可后续扩展。 |
| 运行环境 | 后端以 **Linux** 为主目标；开发可在 Windows 上使用 psutil 获取本机数据，数值与 Linux 生产环境可能不同，部署时需在 Linux 上验证。 |

## 2. 功能需求

### 2.1 必须展示的指标

| 指标 | 说明 | 展示建议 |
|------|------|----------|
| **CPU** | 当前 CPU 使用率（整体或平均） | 百分比；可选简短说明（如「多核平均值」） |
| **内存** | 总内存、已用、可用、使用率 | 总/已用/可用（单位 MB 或 GB）+ 使用率百分比 |
| **磁盘** | 各分区或根分区总容量与已用 | 至少根分区；多分区可列表；总容量、已用、使用率 |
| **网络** | 收发字节数或简要流量状态 | 可展示「累计接收/发送」或「当前速率」；本阶段至少有一项可读指标 |

### 2.2 交互与体验

- **自动刷新**：页面进入后按固定间隔（如 5 秒）请求最新数据，离开页面或切到其他标签时停止轮询。
- **加载与错误**：首次加载显示加载态；接口失败或非 Linux/无权限时展示友好提示，不崩溃。
- **风格**：与现有仪表盘一致（主题变量、卡片式区块、清晰层级）。

### 2.3 进程列表（已实现）

- **进程列表**：在系统监控页当选择「本机」时展示进程列表（PID、名称、状态、用户、内存、CPU）；支持按 CPU/内存/名称/PID 排序、条数限制、名称过滤；仅本机，只读，不提供结束进程等写操作。
- **后续可选**：结束进程、远程进程列表、历史曲线图。

## 3. 接口约定

### 3.1 系统快照接口

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | /api/monitor/stats?target=local\|remote | 需登录；返回指定目标（本机或远程）的 CPU、内存、磁盘、网络快照。本机时 network 可含 rate_sent_kbps、rate_recv_kbps、network_history。 |

**响应示例（结构，具体字段名可微调）：**

```json
{
  "cpu_percent": 12.5,
  "memory": {
    "total_mb": 16384,
    "available_mb": 10240,
    "used_mb": 6144,
    "percent": 37.5
  },
  "disk": [
    {
      "mountpoint": "/",
      "total_gb": 100,
      "used_gb": 45,
      "free_gb": 55,
      "percent": 45
    }
  ],
  "network": {
    "bytes_sent": 1234567890,
    "bytes_recv": 9876543210
  }
}
```

- **cpu_percent**：整体 CPU 使用率（0–100），多核为平均值或聚合值。
- **memory**：总/可用/已用（MB）、使用率（percent）。
- **disk**：数组，每项为挂载点、总/已用/可用（GB）、使用率；至少包含根分区。
- **network**：自开机或某统计周期以来的累计收发字节数；若后端便于实现，可增加「当前速率」等字段。

**错误与降级：**

- 401：未登录，前端跳转登录。
- 500 或依赖不可用：可返回 `{"detail": "系统指标暂不可用"}` 或部分字段为 null，前端展示「暂不可用」或部分数据。

### 3.2 进程列表接口

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | /api/monitor/processes | 需登录；返回本机进程列表（只读）。仅支持本机，远程暂不提供。 |

**查询参数：**

- `limit`：返回条数，默认 50，范围 1–500。
- `sort_by`：排序字段，`cpu_percent`（默认）| `memory_mb` | `name` | `pid`。
- `name_filter`：可选，按进程名模糊过滤。

**响应示例：**

```json
{
  "processes": [
    {
      "pid": 1234,
      "name": "python",
      "status": "running",
      "username": "root",
      "memory_mb": 128.5,
      "cpu_percent": 2.1
    }
  ]
}
```

### 3.3 远程配置与远程安装

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | /api/monitor/remote-config | 需登录；返回是否已配置及 host/port/username（不含密码）。 |
| PUT | /api/monitor/remote-config | 需登录；body `{ host, port, username, password }`，保存并加密存储。 |
| POST | /api/monitor/remote-install-psutil | 需登录；在已配置的远程主机上执行安装 psutil（优先 apt，失败再试 pip）。 |

### 认证

- 与现有鉴权一致：请求头 `Authorization: Bearer <token>`，无 token 或无效返回 401。

## 5. 后端实现要点

- **依赖**：使用 **psutil** 获取 CPU、内存、磁盘、网络（跨平台，Windows 开发 / Linux 部署均可）。
- **路由**：新建 `routers/monitor.py`（或 `system.py`），挂载前缀 `/api/monitor`，在 `main.py` 中注册；`GET /stats` 需依赖 `get_current_user` 做鉴权。
- **数据**：在接口内调用 psutil 读当前值，组装为上述 JSON，不落库；若某指标在当前环境不可用（如无权限读磁盘），可捕获异常并对应字段置 null 或省略，并尽量返回其余可用指标。
- **性能**：单次调用为轻量读系统信息，无需缓存；若后续增加历史数据再考虑缓存或异步。

---

## 6. 前端实现要点

- **页面**：在现有 `/monitor` 页（`views/dashboard/Monitor.vue`）中接入上述接口，移除占位文案。
- **布局**：按区块展示 CPU、内存、磁盘、网络；每块可为卡片（标题 + 数值 + 可选进度条或迷你图）。
- **轮询**：进入页面后 `setInterval` 请求 `/api/monitor/stats`，间隔建议 5 秒；在 `onUnmounted` 或 `onBeforeUnmount` 中清除定时器。
- **状态**：loading（首次）、data（展示）、error（接口失败或不可用）；错误时展示提示文案，不阻塞整页。
- **单位与格式**：内存、磁盘统一为 MB/GB，保留 1–2 位小数；百分比取整或保留 1 位；网络可格式化为 KB/MB/GB 便于阅读。
- **样式**：复用现有 CSS 变量与卡片风格，与概览、侧栏一致。
