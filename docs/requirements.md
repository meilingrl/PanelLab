| 项目 | 说明 |

## 2. 功能需求

### 2.1 用户与登录（高优先级）

- **必须**：用户管理系统、登录/登出（至少账号密码）。
- **进阶（完成度较高时）**：
  - 手机号 + 短信验证码登录（需对接短信服务商）
  - 微信 / QQ 等第三方 OAuth 登录
- **权限**：当前单用户即可；若公网多用户，再考虑角色（如管理员/只读）。

### 2.2 必须功能（MVP）

以下五项缺一不可：

1. **登录**：见上。
2. **仪表盘**：系统概览、快捷入口、基础状态展示。
3. **系统监控**：CPU、内存、磁盘、网络等监控与展示。
4. **网站与反向代理**：站点列表、添加/编辑、Nginx 反向代理配置与生效。
5. **数据库管理**：数据库列表、创建/删除、基础管理（如 MySQL 库与用户）。

### 2.3 可选 / 后续

- 计划任务（cron 管理）
- 多用户与角色
- 安全与加固（防火墙、SSH、日志审计）
- 多服务器管理（面板中台 + 多台被控机）

## 3. 非功能需求

| 维度 | 说明 |
|------|------|
| UI | 美观、清晰，便于日常使用 |
| 部署 | 固定机器；安装方式待定（一键脚本 / Docker / 手动），打包与分发方式待学习 |
| 目标运行环境 | Linux 服务器；开发环境可为 Windows（见 README「开发环境说明」） |

---

## 4. 开发环境与目标环境

- **开发**：可在 Windows 上开发（推荐配合 WSL2 或虚拟机中的 Linux 做真实环境测试）。
- **运行**：软件面向 Linux，涉及系统调用的部分需在 Linux 上验证（路径、权限、换行符等）。
- 详见 README 中「开发环境说明：Windows 开发、Linux 目标」。

## 5. 阶段与优先级（与 README 一致）

| 阶段 | 内容 | 备注 |
|------|------|------|
| 0 | 技术选型与 Hello 面板 | 前后端打通 |
| 1 | 用户体系与仪表盘 | 登录 + 仪表盘骨架 |
| 2 | 系统监控与进程管理 | 监控 + 进程 |
| 3 | 网站与反向代理 | 站点与 Nginx |
| 4 | 数据库与计划任务 | MySQL + cron（计划任务可延后） |
| 5 | 安全与部署、多机扩展（可选） | 部署文档、可选多服务器 |

第三方登录、多服务器等放在主流程稳定后再做。
